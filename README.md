# Awesome Hello Languages [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

> One repository. One greeting. **521** programming languages and data formats,
> grouped by category.

A curated, self-contained collection of **"Hello, World!"** — the same line
written in as many different languages as possible, organized so you can browse
by family instead of scrolling one giant list. Every entry is a real, standalone
file under [`languages/`](languages) that you can open and run (or compile) with
the matching toolchain.

## Repository layout

```
awesome-hello-languages/
├── languages/
│   ├── python/hello.py
│   ├── rust/hello.rs
│   ├── brainfuck/hello.bf
│   └── ...                 # one directory per language/format
├── .gitattributes          # forces LF so Whitespace stays byte-exact
├── LICENSE
└── README.md
```

Each language lives in `languages/<name>/hello.<ext>` and prints exactly
`Hello, World!`.

## Run a few

```bash
python  languages/python/hello.py
ruby    languages/ruby/hello.rb
node    languages/javascript/hello.js
go      run languages/go/hello.go
bash    languages/bash/hello.sh
lua     languages/lua/hello.lua
```

## Notes

- Several esoteric entries are mechanical transliterations of the Brainfuck
  program and are therefore byte-correct by construction: **Ook!**, **Spoon**,
  **Whitespace**, **Alphuck**, **Blub** and **Unlambda**.
- Shader languages (GLSL, HLSL, WGSL, Metal, Cg, ShaderLab, OSL) cannot print
  text — their output is a colour — so the greeting appears in the source as a
  string or comment.
- Some entries are declarative formats (schemas, configs, grammars) where the
  greeting is the value carried by the document.
- Category placement is a judgement call — many languages are multi-paradigm and
  could sit in more than one section.
